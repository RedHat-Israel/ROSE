Command

  • A command is an instruction given by a user telling a computer to do something
  • Linux commands are the essence of the entire Linux operating system used to administer the entire system. You may not know it, but most of the applications you run within the graphical user interface are executing Linux commands for you in the background to accomplish the given task

Directory

  • Actions we can preform on a file are: create, copy, view (read), delete, move file to a different directory, change the file name, run
  • Actions we can preform on a directory are: create, copy, list all files and directories, delete, change your current directory, change name

  • you can also use ~ to get to your home directory
  • Change Directory to the your user home directory

  /home/student
          

  $ cd ~
  $ pwd
  /home/student

Absolute Path vs Relative Path

  • A path is a unique location to a file or a directory in a file system of an OS.
  • An absolute path is defined as the specifying the location of a file or directory from the root directory(/). In other words we can say absolute path is a complete path from start of actual filesystem from / directory.
  • Relative path is defined as path related to the present working directory(pwd).
  • .(a single dot) - this represents the current directory.
  • ..(two dots) - this represents the parent directory.

Absolute path Vs relative path - examples

  • Change to a specific directory- absolute path

  $ cd /home/student/my_test2
  $ pwd
  /home/student/my_test2
  • Move one level up - Relative Path 

  $ cd ..
  $ pwd
  /home/student/
  • Change to a specific directory- relative path

  $ cd my_test2
  $ pwd
  /home/student/my_test2
  • Move two level up - relative path

  $ cd ../..
  $ pwd
  /home

mv

  • Move a file to a different directory
  • The structure of mv command: mv [source] [destination]

  $ mv homework1.py /home/student/my_homeworks/
  $
  • Change a file name

  $ mv homework1.py new_homework1.py
  $

cp

  • Copy a file to a new file
  • The structure of cp command: cp [source] [destination]

  $ cp homework1.py homework1_backup.py
  $
  • Copy a file to a different directory

  $ cp homework1.py /home/student/my_homeworks/
  $
  • Copy a directory and its files into a different directory

  $ cp -r /home/student/my_homeworks/ backups/
  $

rm

  • Remove a file

$ rm homework1_backup.py
$
  • Remove a directory and all its content

  $ rm -r /home/student/backups/
  $
