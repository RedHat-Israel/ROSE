Linux Introduction

Terminal

  • The Terminal is a text-only interface

    • Or move the mouse to the upper left corner of the screen
  • Type "terminal" in the search bar
  • Click on the icon named "Terminal"

Command

  • A command is an instruction given by a user telling a computer to do something
  • Linux commands are the essence of the entire Linux operating system used to administer the entire system
    You may not know it, but most of the applications you run within the graphical user interface are executing Linux commands for you in the background to accomplish the given task

pwd

  • Prints the current directory name

      $ pwd
      /home/student
      $

ls

  • List current directory content

      $ ls
      my_test
      my_test2
      $
  • List specific directory content

      $ ls /home/student
      my_test
      my_test2
      $

ls -l

  • -l is a flag
  • -l print the output in a long listing format

      $ ls -la
      total 8K
      drwxrwxr-x.  2 student student 4.0K Dec  9 22:45 my_test
      drwxrwxr-x.  2 student student 4.0K Dec  9 22:58 my_test2

      $

ls -a

  • -a is a flag
  • -a do not ignore entries starting with 

      $ ls -la
      . .. my_test my_test2
      $

ls -la

  • -la you can use two flags or more at the same time
  • List current directory content with additional information and hidden files

      $ ls -la
      total 12K
      drwxrwxr-x.  3 student student 4.0K Dec  9 22:57 .
      drwxrwxr-x. 10 student student 4.0K Dec  9 22:45 ..
      -rw-rw-r--.  1 student student    0 Dec  9 22:57 .bashrc
      drwxrwxr-x.  2 student student 4.0K Dec  9 22:45 my_test
      drwxrwxr-x.  2 student student 4.0K Dec  9 22:58 my_test2

      $

Directory

  • A directory is just a special file, which contains a list of entries and their ID
  • Actions we can perform on a file are:
    create, copy, view (read), delete, move file to a different directory, change the file name, run
  • Actions we can perform on a directory are:
    create, copy, list all files and directories, delete, change your current directory, change name

mkdir

  • Creates a directory in the current directory

      $ mkdir my_test
      $
  • Creates a directory in a specific directory

      $ mkdir /home/student/my_test2
      $
  • Creates a directory and all parent directories if missing

      $ mkdir -p my_test/dir2/dir3/dir4
      $

cd

  • Change the current directory to another directory

      $ cd my_test
      $

/ – The Root Directory

  • Everything on your Linux system is located under the / directory, known as the root directory

      $ cd /
      $ ls -la
      bin   dev        etc   lib    lost+found  mnt  proc  run   sys  tmp  var
      boot  Downloads  home  media      opt  root  sbin  usr

~ - Home directory

  • Each Linux user has a home directory, this directory is in /home directory
    For example, if you have a user called student, then his default home directory is /home/student
  • You can also use ~ to get to your home directory
  • Change directory to the user home directory:

      $ cd
      $ pwd
      /home/student
              

      $ cd ~
      $ pwd
      /home/student

Absolute Path vs. Relative Path

  • A path is a unique location to a file or a directory in a file system of an OS
  • An absolute path is defined as specifying the location of a file or directory from the root directory(/)
    In other words, we can say that an absolute path is a complete path from start of actual filesystem from / directory
  • Relative path is defined as path related to the present working directory (pwd).
  • . (a single dot) - this represents the current directory
  • .. (two dots) - this represents the parent directory

Absolute Path vs. Relative Path - examples

  • Change to a specific directory - absolute path

      $ cd /home/student/my_test2
      $ pwd
      /home/student/my_test2
  • Move one level up - relative path 

      $ cd ..
      $ pwd
      /home/student/
  • Change to a specific directory - relative path

      $ cd my_test2
      $ pwd
      /home/student/my_test2
  • Move two level up - relative path

      $ cd ../..
      $ pwd
      /home

Vim

  • Vim is an editor to create or edit a text file
  • There are two modes in vim: command mode and insert mode
    • In the command mode, user can move around the file, delete text, etc
    • In the insert mode, user can insert text

Open file with vim

  • To open a file with the vim editor:

      $ vim file-name
      $
  • If the file exists, it will open it with the vim editor
  • If the file does not exists, it will create a new file with this name and open it with the vim editor

Usefull vim commands

  • i - Enter insert mode
  • esc - Exit insert mode
  • :w - Save file
  • :wq - Save and close file
  • :wq! - Force save and close file
  • :q - Close file
  • :q! - Close file, do not> save changes
  • dd - Delete line (cut)
  • yy - Copy line
  • p - Paste
  • gg - Go to the first line
  • G - Go to the last line
  • :n - Go to line n
  • /word - Search word in the file
  • For more usefull commands run: man vim

mv

  • Move a file to a different directory
  • The structure of mv command: mv [source] [destination]

      $ mv homework1.py /home/student/my_homeworks/
      $
  • Change a file name

      $ mv homework1.py new_homework1.py
      $

cp

  • Copy a file to a new file
  • The structure of cp command: cp [source] [destination]

      $ cp homework1.py homework1_backup.py
      $
  • Copy a file to a different directory

      $ cp homework1.py /home/student/my_homeworks/
      $
  • Copy a directory and its files into a different directory

      $ cp -r /home/student/my_homeworks/ backups/
      $

rm

  • Remove a file

    $ rm homework1_backup.py
    $
  • Remove a directory and all its content

      $ rm -r /home/student/backups/
      $

rmdir

  • Remove a directory from the current directory

      $ rmdir my_test
      $
  • Remove a directory and all parent directories

      $ rmdir -p my_test/dir2/dir3/dir4
      $

touch

  • Create new files

      $ touch myfile1.txt myfile2.txt
      $ ls
      myfile1.txt
      myfile2.txt

head

  • Print the first part of files

      $ head -3 ~/.bashrc
      # .bashrc

      # Source global definitions
      $

tail

  • Print the last part of files

      $ tail -3 ~/.bashrc
      # export SYSTEMD_PAGER=

      # User specific aliases and functions
      $

clear

  • Clear the terminal screen

      $ clear
      $
